핵심 메시지:

- 비전문가도 이해할 수 있는 언어로 설명해봤다
- 문서화, 발표, IR도 내 일의 일부였다

## 16. Closing
현재 문구도 괜찮지만 더 강하게 다듬어라.

예시:
- 관심에서 시작해 경험으로 증명하고 습관으로 굳혔습니다
- 작은 실행을 쌓아 더 큰 결과를 만들겠습니다
- 기술을 연결해 실제로 작동하는 결과를 만들겠습니다

[디자인 가이드]
- 개발자 포트폴리오 grid 느낌 금지
- 슬라이드형 스크롤
- 한 화면 한 메시지
- 여백 넓게
- 텍스트 짧게
- 제목은 크고 강하게
- body는 짧고 dense하지 않게
- keyword / thinking / contribution / photo / related url은 메인 프로젝트 공통 구조로 통일
- project-body 줄글은 줄이고, “한 줄 결과”를 새로 강조
- 관련 URL은 chips 스타일 또는 작은 link row로 새로 추가
- tech-grid는 보조 정보로만 사용하고, keyword보다 약하게 보여라

[구현 방식]
- 현재 index.html 직접 리팩토링
- 새로 필요한 작은 CSS 클래스만 추가
- 기존 inline style은 가능하면 정리
- 필요하면 data/projects.js에 mainProjectSchema를 만들고 텍스트를 구조화
- 단, 완전한 프레임워크 전환 금지
- 정적 배포 유지
